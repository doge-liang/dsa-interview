## 常用数学公式

### 复杂度增长速度（从快到慢）

O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

### 对数公式

- log₂(2ⁿ) = n
- log₂(n) ≈ 3.32 log₁₀(n)
- logₐ(b) = logₙ(b) / logₙ(a)

### 级数求和

- Σ₁ⁿ i = n(n+1)/2 = O(n²)
- Σ₁ⁿ i² = n(n+1)(2n+1)/6 = O(n³)
- Σ₁ⁿ 2ⁱ = 2ⁿ⁺¹ - 1 = O(2ⁿ)

### 等比数列

- a + ar + ar² + ... + arⁿ⁻¹ = a(1 - rⁿ) / (1 - r)

### 主定理（Master Theorem）

对于递归关系 T(n) = aT(n/b) + f(n)：

1. 如果 f(n) = O(n^c)，其中 c < log_b(a)
   - T(n) = O(n^{log_b(a)})

2. 如果 f(n) = O(n^{log_b(a)} log^k n)，其中 k ≥ 0
   - T(n) = O(n^{log_b(a)} log^{k+1} n)

3. 如果 f(n) = O(n^c)，其中 c > log_b(a)
   - T(n) = O(n^c)

## 空间复杂度分类

- O(1)：常量空间（只使用固定数量的变量）
- O(n)：线性空间（使用与输入规模成比例的空间）
- O(log n)：对数空间（如递归深度）
- O(n²)：平方空间（如动态规划二维数组）

## 时间复杂度估算经验

- 10⁸ 次操作 ≈ 1 秒（现代 CPU）
- 10⁹ 次操作 ≈ 10 秒
- 通常时间限制：1-2 秒，操作次数在 10⁷-10⁸ 之间

## 常见优化技巧

1. **空间换时间**：使用哈希表加速查找
2. **预处理**：提前计算并存储结果
3. **二分优化**：将 O(n) 优化为 O(log n)
4. **滑动窗口**：减少重复计算
5. **双指针**：减少循环嵌套
6. **位运算**：加速某些数学运算

## 快速查找技巧

- 查找最大值/最小值：O(n)
- 查找第 K 大元素：O(n) 平均，O(n log n) 最坏（堆排序）
- 判断元素存在性：O(1) 哈希表，O(log n) 平衡树
- 去重：O(n) 哈希表，O(n log n) 排序

## 注意事项

1. **时间复杂度**：关注最坏情况
2. **空间复杂度**：关注额外空间（不包括输入）
3. **实际性能**：考虑缓存、常数因子
4. **适用场景**：根据数据特点选择合适算法
